## 1. 공통 문제

### 1.1 브라우저에서 UI가 보이지 않음

**문제**: 개발 서버 실행 후 브라우저에 아무것도 표시되지 않음

**해결방안**:
1. 개발 서버 상태 확인: `npm run dev` 실행 확인
2. 포트 확인: `http://localhost:3000` 접속 확인
3. 브라우저 콘솔 에러 확인: F12 개발자 도구에서 에러 메시지 확인
4. 빌드 확인: `npm run build` 실행하여 빌드 에러 확인
5. 캐시 클리어: `.next` 폴더 삭제 후 재실행

**방지책**:
- 개발 서버 실행 시 에러 메시지를 항상 확인
- 브라우저 콘솔을 정기적으로 확인하는 습관화
- 주요 변경사항 후 빌드 테스트 수행

---

### 1.2 컴포넌트를 어디에 둬야 할지 모르겠음

**문제**: 새 컴포넌트를 만들 때 적절한 위치를 결정하기 어려움

**해결방안**:
1. `/doc/ARCHITECTURE.md`의 "2. 컴포넌트 분류 및 관리" 섹션 참조
2. 컴포넌트 분류 의사결정 트리 사용:
   - 레이아웃 요소? → `components/layout/`
   - 재사용 가능? → `components/ui/`
   - 페이지 전용? → `app/pages/{route}/_components/`

**방지책**:
- ARCHITECTURE.md 문서를 정기적으로 참조
- 컴포넌트 생성 전 분류 기준 확인
- 팀 내 컴포넌트 분류 기준 공유

---

### 1.3 스타일이 적용되지 않음

**문제**: Tailwind CSS 클래스를 적용했지만 스타일이 반영되지 않음

**해결방안**:
1. `tailwind.config.ts`의 `content` 경로 확인
2. 개발 서버 재시작: Tailwind 설정 변경 시 필요
3. 클래스명 오타 확인: Tailwind 클래스명 정확성 확인
4. CSS 파일 import 확인: `globals.css` import 확인

**방지책**:
- Tailwind 설정 변경 시 문서화
- 일반적인 Tailwind 패턴 사용 (임의 값 지양)
- 스타일 변경 후 빌드 확인

## 2. 빌드 및 환경 문제

### 2.1 TypeScript 타입 에러

**문제**: TypeScript 컴파일 시 타입 관련 에러 발생

**해결방안**:
1. 타입 정의 확인: `lib/types/` 폴더의 타입 정의 확인
2. 타입 명시: `any` 타입 사용 지양, 명시적 타입 지정
3. `tsconfig.json` 설정 확인: 타입 체크 설정 확인

**방지책**:
- 타입을 명시적으로 정의하는 습관
- `any` 타입 사용 최소화
- 정기적인 타입 체크 실행

---

### 2.2 의존성 설치 문제

**문제**: `npm install` 실행 시 에러 발생

**해결방안**:
1. Node.js 버전 확인: 프로젝트 요구 버전 확인
2. `package-lock.json` 삭제 후 재설치
3. `node_modules` 폴더 삭제 후 재설치
4. 캐시 클리어: `npm cache clean --force`

**방지책**:
- Node.js 버전 명시 (`.nvmrc` 파일 사용)
- 패키지 추가 시 버전 명시
- 의존성 변경사항 문서화

---

## 3. 아키텍처 및 구조 문제

### 3.1 폴더 구조 규칙 위반

**문제**: ARCHITECTURE.md에 정의된 폴더 구조와 다른 위치에 파일 생성

**해결방안**:
1. ARCHITECTURE.md의 "1. 폴더 구조" 섹션 참조
2. 파일을 올바른 위치로 이동
3. import 경로 수정

**방지책**:
- 파일 생성 전 ARCHITECTURE.md 확인
- 코드 리뷰 시 폴더 구조 검증
- 팀 내 아키텍처 가이드 공유

---

### 3.2 컴포넌트 재사용 불가

**문제**: 특정 페이지에만 사용하는 컴포넌트가 재사용이 필요해짐

**해결방안**:
1. 컴포넌트를 `app/pages/{route}/_components/`에서 `components/ui/`로 이동
2. Props를 범용적으로 수정
3. 모든 import 경로 업데이트

**방지책**:
- 컴포넌트 설계 시 재사용 가능성 고려
- Props를 범용적으로 설계하는 습관
- 컴포넌트 이동 프로세스 문서화

---

## 4. 스타일 및 디자인 시스템 문제

### 4.1 디자인 토큰이 적용되지 않음

**문제**: Figma 디자인 토큰을 사용했지만 스타일이 다름

**해결방안**:
1. ARCHITECTURE.md의 "5. 기술 실행 규칙" 섹션 참조
2. CSS 변수 정의 확인: `globals.css`의 `:root` 변수 확인
3. Tailwind 설정 확인: `tailwind.config.ts`의 토큰 매핑 확인
4. 토큰 값 검증: 실제 사용 값과 정의된 토큰 값 비교

**방지책**:
- 토큰만 사용 (임의 값 지양)
- 토큰 추가 시 문서화
- 정기적인 디자인 토큰 검증

---

### 4.2 반응형 디자인 문제

**문제**: 반응형 스타일이 예상대로 동작하지 않음

**해결방안**:
1. ARCHITECTURE.md의 "5.2 반응형 디자인 필수 규칙" 섹션 참조
2. Breakpoint 기준 확인: Tailwind 기본 breakpoint 사용
3. 반응형 패턴 확인: 공통 반응형 패턴 사용
4. 브라우저 개발자 도구에서 실제 화면 크기 확인

**방지책**:
- 공통 반응형 패턴 사용
- 임의 breakpoint 정의 지양
- 다양한 화면 크기에서 테스트
